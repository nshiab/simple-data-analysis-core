# Phase 4 results (issue #78)

Internal reorganization — no user-visible change, no benchmark run needed.

## One convention everywhere

`SimpleTable` and `SimpleDB` class methods are now thin, JSDoc'd wrappers; every
implementation lives in `src/methods/`. Neither class file runs a query inline
anymore (the two exceptions are `SimpleDB.customQuery` and
`SimpleDB.getExtensions`, whose bodies are already single calls to the `queryDB`
helper).

- 92 methods extracted from `SimpleTable`, 4 from `SimpleDB`.
- All 48 `*Query.ts` string builders are gone as separate files: each is a
  private function inside the one method file that uses it.
  `removeColumnsQuery.ts` was imported by nothing and was deleted.
- `SimpleTable.ts`: 1,700 lines of actual code (7,215 total before the refactor;
  the ~4,100 remaining non-code lines are JSDoc, which stays on the class so
  editor hovers and generated docs keep working).

## Verification

- Full test suite green after each batch (913 tests).
- `llm.md` regenerated by `deno task all-tests`: byte-identical, confirming the
  `deno-docs-to-md` output is unaffected by the moves.
